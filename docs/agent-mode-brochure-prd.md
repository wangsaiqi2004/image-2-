## 2. 背景与问题

当前 Image Studio 的能力已经覆盖：

- 参考图上传
- 提示词输入
- 参数配置
- 批量生图
- 发送前分析
- 行业 Agent
- 本地图库
- 管理后台日志

但用户在真实使用中仍然有三个明显问题：

1. 参数负担重  
用户需要自己决定张数、宽高比、分辨率、质量、提示词是否拆分，尤其是多图任务和宣传画册任务时，决策成本过高。

2. 多图需求无法自然表达  
用户经常会说“帮我做 4 张不同方向的海报”或“做一本 8 页公司宣传彩页”，但当前工作流更像是一次只提交一批同构任务，不适合处理“每张图都不一样”的自然语言需求。

3. 宣传画册场景缺少完整工作流  
画册不是单图问题，而是“整本方向探索 -> 选模板 -> 单页精修 -> 整本导出”的连续过程。直接一页页生成，成本高、风格散、返工重。

因此需要新增一个更高层的产品入口：`Agent 模式 A`。

## 3. 产品定义

`Agent 模式 A` 是 Image Studio 的顶层智能入口。用户打开开关后，界面简化为“上传参考图 + 输入自然语言需求 + 发送”，系统先理解用户意图，再自动决定后续执行路径。

`Agent 模式 A` 在 MVP 阶段支持 3 类任务：

1. 单图生成  
用户描述一张图，系统自动识别适合的张数、比例、尺寸、分辨率、提示词优化方式，并自动进入生成。

2. 多图异构生成  
用户一次描述多张不同图片，系统自动拆成多条任务，每张图片拥有自己的提示词、参数和参考图使用策略。

3. 宣传画册 / 彩页生成  
用户通过自然语言描述“公司宣传画册”需求，系统自动进入画册子流程，先生成整本模板板，再展开为单页精修。

## 4. 核心目标

### 4.1 业务目标

- 提升首次可用性，让不懂提示词和参数的用户也能完成任务
- 提升多图任务和宣传画册任务的转化率
- 降低无效生成次数，减少用户试错成本
- 把 Image Studio 从“生图工具”升级为“自然语言视觉任务工作台”

### 4.2 用户目标

- 用户只说需求，不需要先想清楚每个参数
- 用户提出多图需求时，不需要自己手工拆单
- 用户做宣传画册时，不需要一页页从零摸索
- 用户需要改单页时，可以只改那一页，不重跑整本

### 4.3 非目标

- 本期不承诺直接生成可印刷的最终可编辑画册源文件
- 本期不处理长篇稳定文字排版问题
- 本期不支持多人协作、审批流、品牌中心
- 本期不替代当前标准模式，标准模式仍保留给高级用户

## 5. 用户角色

| 角色 | 典型身份 | 核心任务 | 主要痛点 |
| --- | --- | --- | --- |
| 轻度用户 | 小老板、运营、行政 | 做宣传图、简单海报 | 不懂参数和提示词 |
| 中度用户 | 市场、品牌、销售支持 | 一次做多张不同物料 | 不会拆任务，不想重复录入 |
| 重度用户 | 设计师、代运营、创意团队 | 做整套宣传画册、提案、比稿 | 需要整本一致性和后续改单页 |

## 6. 典型使用场景

### 6.1 单图场景

用户输入：

```text
帮我做一张公司官网头图，AI 数据分析平台，科技感，但不要太虚，要像 B2B SaaS 首页主视觉。
```

系统应输出：

- 识别任务类型：`single_image`
- 自动推荐比例：`16:9`
- 自动推荐分辨率：`2K`
- 自动生成 2 到 4 张变体
- 自动优化提示词并进入生成

### 6.2 多图异构场景

用户输入：

```text
帮我做 4 张图：
第 1 张做官网头图，科技蓝；
第 2 张做微信封面，偏商务；
第 3 张做产品功能海报，突出数据大屏；
第 4 张做招聘海报，要更年轻一点。
```

系统应输出：

- 识别任务类型：`multi_image_batch`
- 自动拆成 4 个子任务
- 每个子任务拥有独立提示词、比例、尺寸、质量、风格说明
- 默认先展示“任务拆解结果”
- 用户确认后统一提交到批量队列

### 6.3 画册场景

用户输入：

```text
帮我做一个智能制造公司的 8 页宣传画册，包含封面、公司介绍、核心优势、产品、案例、团队、联系方式和封底。先给我 4 套不同风格的整本模板，我选一个之后再细化每一页。
```

系统应输出：

- 识别任务类型：`brochure_project`
- 自动生成页结构
- 自动生成 4 套整本模板板
- 用户选中 1 套后再生成单页
- 支持单页重做、整本预览、导出

## 7. 产品方案总览

### 7.1 双模式结构

- `标准模式`
  - 保留当前工作台逻辑
  - 用户可以手动控制模型、比例、分辨率、张数、提示词分析和行业 Agent

- `Agent 模式 A`
  - 用户只保留核心输入能力
  - 系统自动理解意图并编排任务
  - 标准参数区默认隐藏
  - 当前“行业 Agent”降级为二级能力，不再作为顶级主入口

### 7.2 Agent 模式 A 的执行路径

1. 用户开启 `Agent 模式 A`
2. 用户上传参考图并输入自然语言需求
3. 点击发送
4. 系统先做意图识别和任务拆解
5. 系统根据任务类型执行：
   - 单图：高置信度时自动生成
   - 多图：展示拆解结果，用户确认后生成
   - 画册：先生成模板板，再走画册子流程
6. 所有结果进入当前图库和管理后台日志

## 9. 交互设计

## 9.1 开关位置

### 推荐位置

将 `Agent 模式 A` 的开关放在当前工作台输入区顶部，也就是现有 `agent-quickbar` 所在区域的最右侧，形成一条新的 `composer-mode-bar`。

原因：

- 用户进入工作台后的视觉焦点就是输入区
- Agent 模式属于“任务输入方式”的切换，而不是“参数设置”的切换
- 放在输入区顶部，能在不打断当前布局的前提下，清晰区分“标准模式”和“Agent 模式”
- 不应放入右侧设置面板，因为那会让这个能力看起来像一个参数，而不是一个工作流入口

### 结构建议

- 左侧：保留轻量说明，例如 `当前模式：标准 / Agent`
- 右侧：`Agent 模式 A` pill switch

## 9.2 开关样式

### 样式定义

- 形式：胶囊开关 + 标签
- 尺寸：`56 x 30` 开关主体，外层整体高度 `34`
- 圆角：`999px`
- 关闭态：
  - 背景：浅灰白
  - 描边：`var(--border)`
  - 文案：`Agent`
  - 辅助说明：`自动编排`
- 开启态：
  - 背景：青绿色到浅青色渐变
  - 描边：弱化
  - 拨钮带 `Sparkles` / `Bot` 图标
  - 外发光：轻微 focus ring
  - 文案：`Agent 模式 A`
  - 辅助说明：`理解任务并自动拆解`

### 状态反馈

- `off`：标准模式
- `on-idle`：已启用，等待输入
- `on-analyzing`：显示微弱扫描动效
- `on-executing`：显示执行状态点
- `on-needs-confirmation`：显示 amber 边框，提示“待确认”

## 9.3 Agent 模式 A 开启后的界面规则

开启后，工作台输入区只保留以下核心区域：

- 上传参考图按钮
- 参考图条带
- 提示词输入框
- 内嵌发送按钮
- Agent 执行状态条

默认隐藏以下区域：

- 当前 `行业 Agent quickbar`
- 预设提示词按钮
- 右侧参数配置按钮
- 发送前优化切换
- prompt assist bar
- 标准模式的参数元信息

说明：

- “只有上传参考图和输入提示词的地方”在交互上应理解为“对用户可见的主控件极简化”
- 发送动作仍然保留，但作为输入区内部的一部分，不再表现为参数驱动操作

## 9.4 输入区原型说明

### 标准模式

沿用当前布局：

- 上传按钮
- 预设按钮
- 提示词框

- 第二行：参考图缩略条

- 发送按钮固定在输入框右下角
- 状态条在输入框下方折叠展示

## 10. Agent 模式 A 的任务理解逻辑

## 10.1 意图识别分类

系统首先输出一个 `intent_type`：

- `single_image`
- `multi_image_batch`
- `brochure_project`
- `page_refine`
- `unknown`

## 10.2 输入理解维度

系统需要从自然语言中提取：

- 用户要生成几张图
- 图片之间是否同构
- 每张图是否有独立用途
- 是否需要多页画册
- 是否明确指定尺寸、比例、分辨率、风格
- 是否引用了参考图
- 是否包含页面级修改指令

## 10.3 任务路由规则

### 规则 1：单图任务

满足条件：

- 只描述一个明确视觉目标
- 没有“第 1 张 / 第 2 张 / 分别 / 各自”等拆分信号
- 没有页码或画册结构词

执行策略：

- AI 自动优化提示词
- 自动推断参数
- 高置信度时直接提交生成

### 规则 2：多图异构任务

满足条件：

- 输入中出现多个独立素材目标
- 或出现“做 3 张 / 4 张 / 分别是 / 每张不一样”

执行策略：

- 先拆成 `N` 个子任务
- 为每个子任务生成单独提示词和参数
- 默认展示一份“任务拆解面板”
- 用户确认后再统一提交

### 规则 3：宣传画册任务

满足条件：

- 包含“画册 / 彩页 / 宣传册 / brochure / 封面 / 第几页 / 联系方式页”等结构化页面信号

执行策略：

- 进入画册子流程
- 先做页结构规划
- 再做模板板探索
- 选定模板后再做单页生成

### 规则 4：改单页任务

满足条件：

- 用户在已有画册项目上下文中提及“第 3 页改一下”“封面再商务一点”

执行策略：

- 定位目标页
- 生成局部修改任务
- 只重跑当前页

## 10.4 自动执行与确认规则

为了兼顾效率和成本，Agent 模式 A 不采用“所有任务都直接自动跑”的激进策略，而是按成本分级。

### 自动执行

满足以下全部条件时自动执行：

- `single_image`
- 置信度 >= `0.85`
- 预计生成张数 <= `4`
- 没有明显冲突参数
- 没有高成本画册流程

### 需要确认

满足以下任一条件时先确认：

- `multi_image_batch`
- `brochure_project`
- 预计生成张数 > `4`
- 任务拆解置信度 < `0.85`
- 存在参数冲突
- 预计消耗较大

### 用户体验规则

- 自动执行前显示 `1.5s` 轻提示：`已理解需求，正在自动提交`
- 用户可以在该提示期点击“停止”
- 需要确认时展示结构化任务卡片，不自动提交

## 10.5 参数决策优先级

为避免 Agent 自动推断和用户明确要求冲突，参数采用如下优先级：

1. 用户在自然语言中明确指定  
例如“做 3 张”“16:9”“4K”“竖版封面”。

2. 任务类型默认规则  
例如官网头图默认优先 `16:9`，微信公众号封面默认优先横版安全区，宣传画册封面默认优先纵版。

3. 参考图约束  
若用户上传了明显的品牌 KV、产品图或版式参考图，系统应优先继承其画面气质和构图方向。

4. 系统推荐值  
当以上都缺失时，才由 Agent 根据历史规则自动补全。

补充规则：

- 若用户明确指定的参数不被当前协议支持，Agent 不得静默覆盖，应在确认面板中标明“已替换为最近似可用值”
- 若用户未指定张数，单图默认 `2` 张变体，多图任务每个子任务默认 `1` 到 `2` 张，画册模板板默认 `4` 套
- 若用户未指定分辨率，单图默认 `2K`，多图默认按用途自动推断，画册模板板阶段默认低分辨率，单页展开默认 `2K`

## 11. 宣传画册子流程

## 11.1 目标

将“生成公司彩页 / 宣传画册”从一次性生图升级为可控的项目型流程。

## 11.2 子流程

1. 画册规划
2. 模板板生成
3. 模板选择
4. 单页展开
5. 单页精修
6. 整本预览
7. 导出

## 11.3 画册规划阶段

系统抽取：

- 公司名称
- 行业
- 用途
- 目标受众
- 页数
- 是否含封面和封底
- 品牌色
- 品牌风格
- 是否需要案例页、产品页、团队页、联系方式页

系统生成：

- `brochure_outline`
- 每页职责
- 推荐模板风格组

## 11.4 模板板阶段

模板板是成本控制核心。

### 输出要求

- 一张图内包含整本所有页面缩略布局
- 一次给出 `4` 套不同风格
- 每套模板需体现：
  - 封面视觉
  - 内页结构节奏
  - 色彩系统
  - 图像语言
  - 信息密度

### 价值

- 先选方向，再展开细页
- 降低整本误生成成本
- 让用户更容易做提案和选择

## 11.5 单页展开阶段

用户选中模板后：

- 系统为每一页生成独立 `page_spec`
- 每页继承整本风格基因
- 按页职责分别生成高质量页面图

## 11.6 单页精修阶段

支持以下操作：

- 保留风格，重写内容
- 保留内容，重做风格
- 封面更商务
- 第 4 页产品更清晰
- 第 6 页案例感更强

## 12. 功能需求明细

## 12.1 模式切换

### 功能说明

- 用户可以在工作台输入区顶部开启或关闭 `Agent 模式 A`
- 模式切换立即生效
- 切回标准模式后，保留用户最近一次手动参数设置

### 规则

- 已上传参考图不丢失

- `reasoning_summary`

- `jobs`
- `brochure_project`

## 12.7 现有行业 Agent 的继承策略

当前产品内已有“行业 Agent”能力。为避免新旧 Agent 概念冲突，MVP 采用以下策略：

- `Agent 模式 A` 作为顶层入口
- 现有 `行业 Agent` 改名为 `行业模板`
- 在标准模式下，`行业模板` 保留原能力
- 在 Agent 模式 A 下，`行业模板` 不作为显式主入口展示，只作为二级增强器使用

触发方式：

- 当 Agent 识别到非常明确的行业场景时，可内部复用行业模板的提示词骨架和默认负面词
- 用户不需要理解“行业 Agent”和“Agent 模式 A”的区别

设计目标：

- 对用户只暴露一个智能入口
- 对系统内部保留现有资产复用

## 20. 边界与异常处理

## 20.1 意图不明确

策略：

- 返回 `unknown`
- 展示“我理解到 2 种可能任务”的澄清面板
- 不直接消耗生图次数

## 20.2 多图拆解冲突

例如用户说“做 4 张图，每张都不一样，但统一成 1:1 和 16:9”。

策略：

- 标注冲突项
- 让用户选择统一比例还是每张独立比例

## 20.3 画册页数异常

例如输入“做一本 40 页画册”。

策略：

- 提示超出 MVP 推荐页数
- 推荐拆成多个章节项目
- 或先生成前 8 到 12 页概念版

## 20.4 参考图不足

策略：

- 若参考图无法支撑品牌一致性，仅给风险提示，不阻断流程

## 24. 结论

这不是单纯加一个开关，而是把 Image Studio 从“参数驱动型生图工具”升级为“自然语言驱动的视觉任务编排器”。

`Agent 模式 A` 是顶层入口，解决：

- 单图更省心
- 多图更会拆
- 画册更成体系

而宣传画册工坊则是 Agent 模式下最有商业价值的高阶场景，它能把整本物料生成这件事变成可探索、可选择、可改单页、可复用的项目流程。
